# Time-Based Profile Unlock Feature - Implementation Summary

## Overview
A complete time-based profile unlock feature has been implemented for locked patients with payment support. Patients can now purchase unlock plans (1, 3, or 6 months) to regain full access to their profiles.

### 2. **Backend Logic - Unlock Expiry Check**
**File:** [adminapp/utils.py](adminapp/utils.py)

**Updated Function:** `ensure_patient_lock_status(patient)`

**Key Features:**
- Checks if unlock period has expired (`date.today() > unlock_end_date`)
- Automatically reverts profile to LOCKED status when unlock expires
- Sends notification when unlock expires
- Validates unlock dates before allowing operations
- Maintains backward compatibility with existing lock logic

**Logic Flow:**
```
1. Check if unlock_end_date exists and has expired
   → If YES: Revert to LOCKED + Send expiry notification
2. Check if currently UNLOCKED and unlock_end_date is valid
   → If YES: Return FALSE (not locked)
3. Proceed with existing lock logic (6 months after delivery)
```

### 3. **Profile Unlock View**
**File:** [patientapp/views.py](patientapp/views.py#L210-L274)

**New Function:** `profile_unlock(request)`

**Features:**
- Displays 3 unlock plans:
  - 1 Month: ₹100
  - 3 Months: ₹250
  - 6 Months: ₹450
- Handles payment simulation (ready for real payment gateway integration)
- Sets unlock dates based on selected plan
- Creates notifications for patient and admin on successful payment
- Shows unlock expiry information when already unlocked

**Payment Flow:**
```python
UNLOCK_PLANS = {
    '1': {'months': 1, 'price': 100, 'label': '1 Month'},
    '3': {'months': 3, 'price': 250, 'label': '3 Months'},
    '6': {'months': 6, 'price': 450, 'label': '6 Months'},
}

On successful payment:
- profile_lock_status = 'unlocked'
- unlock_start_date = today()
- unlock_end_date = today() + (months * 30 days)
```

## Feature Capabilities

### While Profile is UNLOCKED and Current Date ≤ Unlock End Date:
✓ Patient can upload medical records
✓ Patient can book appointments
✓ Doctor can prescribe and add consultation notes
✓ Doctor can add doctor notes to medical records

### When Unlock Duration Expires:
✓ Profile automatically reverts to LOCKED status
✓ Patient receives expiry notification
✓ All locked restrictions apply again

### Backend Enforcement:

✓ Automatic date comparison: `date.today() > unlock_end_date`

## Payment Integration Points

The implementation uses a **demo payment system** that can be easily replaced with real payment gateways:

**Demo Flow:**
1. User selects plan
2. User clicks "Proceed to Payment"
3. Mock payment alert shown
4. Payment status set to 'success'
5. Form submitted with payment_status = 'success'

**To Integrate Real Payment (e.g., Razorpay):**

```python
# In profile_unlock view
if plan_id in UNLOCK_PLANS and payment_status == 'success':
    # (Real payment verification here)
    # Set unlock dates and save
```

Add in template:
```html
<script src="https://checkout.razorpay.com/v1/checkout.js"></script>
<!-- Razorpay integration code -->
```

## Notification System

Notifications are automatically sent for:

1. **Profile Locked:** When 6 months after delivery
   - Message: "Your profile has been locked... You can unlock your profile with a payment plan."

2. **Unlock Purchased:** After successful payment
   - Patient: "Your profile has been unlocked for X Month(s). Your unlock will expire on [DATE]."
   - Admin: "Patient [NAME] purchased X Month(s) unlock for ₹[PRICE]."

3. **Unlock Expired:** When unlock period ends
   - Message: "Your unlock period has expired. Your profile is now locked again."
